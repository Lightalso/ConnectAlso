## 一、系统概述

ConnectAlso 是一个跨平台虚拟局域网工具，包含以下核心组件：

```
┌──────────────┐     ┌───────────────┐     ┌──────────────┐
│  控制服务     │◄───►│   协调服务     │◄───►│  STUN 服务   │
│  (Control)   │     │ (Coordinator)  │     │   (STUN)     │
└──────┬───────┘     └───────────────┘     └──────────────┘
       │
       │ HTTPS / WSS
       ▼
┌──────────────┐  P2P (优先) / Relay (降级)  ┌──────────────┐
│   客户端 A    │◄────────────────────────────►│   客户端 B    │
│  (Daemon)    │     加密 UDP 隧道              │  (Daemon)    │
└──────┬───────┘                               └──────┬───────┘
       │                                              │
       ▼                                              ▼
   ┌──────┐                                      ┌──────┐
   │ TUN  │                                      │ TUN  │
   └──────┘                                      └──────┘
```

**信任边界：**
1. 客户端 ↔ 控制服务 (TLS/HTTPS)
2. 客户端 ↔ 客户端 (端到端加密)
3. 中继服务 (密文转发，不参与解密)
4. STUN 服务 (无状态，仅返回地址)
5. 本地设备 (TUN + 私钥存储)

## 二、资产清单

| 资产 | 敏感度 | 存储位置 | 威胁 |
|------|--------|----------|------|
| 设备长期私钥 (X25519) | **极高** | 系统安全存储 (Keychain/Keystore/DPAPI) | 泄露 = 身份冒充 |
| 会话密钥 (临时 DH) | **高** | 内存中，会话结束后清除 | 泄露 = 会话解密 |
| 虚拟网络流量 | **高** | TUN 接口 + 加密隧道 | 窃听 = 流量分析/数据泄露 |
| 设备身份 (UUID/PubKey) | **中** | 控制服务数据库 + 设备本地 | 伪造 = 未授权加入网络 |
| 控制面配置 (ACL、路由) | **中** | 控制服务数据库 | 篡改 = 网络拓扑操控 |
| 中继节点列表 | **低** | 控制服务下发 | 窃取 = 了解基础设施 |
| STUN 响应 | **无** | 公共信息 | 无 |

## 三、威胁行为者

| 行为者 | 能力 | 动机 | 可能性 |
|--------|------|------|--------|
| **被动网络窃听者** | 观察公网流量、分析流量模式 | 情报收集、隐私侵犯 | 高 |
| **主动中间人 (MITM)** | 拦截/篡改控制面通信 | 身份冒充、配置篡改 | 中（TLS 防护） |
| **恶意中继节点** | 分析/篡改/丢弃转发的密文 | 流量分析、服务中断 | 中（自托管模式下低） |
| **恶意网络参与者** | 利用已加入的网络进行攻击 | 横向移动、资源滥用 | 中 |
| **设备失窃/丢失** | 物理访问本地设备 | 提取私钥、访问虚拟网络 | 低-中 |
| **恶意控制服务管理员** | 完全控制控制面 | 监控、操控全网 | 低（自托管） |
| **DoS 攻击者** | 发送大量无效包 | 服务中断 | 高 |

## 四、逐组件威胁分析

### 4.1 密钥生成与存储

| 威胁 | 描述 | 缓解 |
|------|------|------|
| T1: 弱随机数 | 密钥生成使用弱熵源 | 使用 `getrandom` (OS CSPRNG)，拒绝 `rand::thread_rng` 用于密钥生成 |
| T2: 私钥泄露 | 设备长期私钥被其他进程读取 | macOS Keychain / Android Keystore / Windows DPAPI / Linux `keyutils` |
| T3: 内存转储 | 会话密钥从内存中提取 | `zeroize` crate 在会话结束时清零密钥；考虑 `mlock` 防换出 |

### 4.2 控制面通信

| 威胁 | 描述 | 缓解 |
|------|------|------|
| T4: TLS 降级 | 强制降级到弱密码套件 | TLS 1.3 仅；固定密码套件；证书固定 (HPKP) |
| T5: 控制服务冒充 | 伪造控制服务身份 | TLS 证书验证；客户端可配置固定公钥 |
| T6: 未授权设备注册 | 攻击者注册恶意设备 | 邀请码机制（可过期、限次）；管理员审批 |
| T7: 重放攻击 | 重放控制面请求 | HTTP/2 内置重放保护；请求含时间戳 + nonce |

### 4.3 数据面 (P2P 隧道)

| 威胁 | 描述 | 缓解 |
|------|------|------|
| T8: 密文篡改 | 攻击者修改传输中的密文 | ChaCha20-Poly1305 AEAD：篡改导致认证失败，包丢弃 |
| T9: 重放攻击 | 重放旧的加密包 | 单调递增 nonce；`SessionCipher` 拒绝过期 nonce（M2 实现） |
| T10: 侧信道 | 包时序/大小泄露信息 | 可选：固定长度填充；M3 后评估 |
| T11: 协议降级 | 强制使用弱算法 | 硬编码加密套件；禁止协商 |
| T12: Nonce 复用 | 同一密钥重复使用 nonce | 发送/接收方向使用不同 nonce 空间（`RESPONDER_TX_OFFSET = 2^32`） |
| T13: DH 密钥泄露 | 临时私钥从内存泄露 | 会话结束后清零；短期会话 |

### 4.4 中继服务

| 威胁 | 描述 | 缓解 |
|------|------|------|
| T14: 流量分析 | 中继观察通信模式 | 中继仅见密文+帧头；发送节奏扰乱（M2 后评估） |
| T15: 中继篡改 | 中继修改密文 | 端到端 AEAD：篡改导致客户端解密失败 |
| T16: 未授权中继使用 | 攻击者消耗中继带宽 | 客户端身份验证（设备注册）；速率限制 |
| T17: 中继 DoS | 大量垃圾包淹没中继 | UDP 限流；HELLO 注册验证 |

### 4.5 NAT 穿越

| 威胁 | 描述 | 缓解 |
|------|------|------|
| T18: STUN 欺骗 | 伪造 STUN 响应改变打洞目标 | 仅使用已知/可信 STUN 服务器 |
| T19: 打洞滥用 | 攻击者利用打洞包进行端口扫描 | 打洞包仅发送到已知对端，限制重试次数 |

### 4.6 平台适配层 (TUN)

| 威胁 | 描述 | 缓解 |
|------|------|------|
| T20: 未授权 TUN 访问 | 本地恶意进程读写 TUN | 文件权限 (Unix) / 进程 ACL (Windows) |
| T21: 路由劫持 | TUN 接口路由被篡改 | 监控路由表变化；最小权限路由 |

## 六、安全开发原则

1. **不自行设计密码学原语**：使用经审计的库和标准协议
2. **最小权限**：每个组件仅拥有完成任务所需的最小权限
3. **纵深防御**：多层防护（TLS + AEAD + ACL）
4. **默认安全**：不安全配置需显式操作
5. **可审计**：`unsafe_code = "deny"` (lints 已配置)
6. **私钥不离设备**：长期私钥仅本地生成和保存
7. **中继不参与解密**：中继只转发密文
8. **日志不含敏感信息**：不含私钥、token、业务流量内容
