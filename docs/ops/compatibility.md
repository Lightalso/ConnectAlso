# ConnectAlso 发布兼容策略与长期维护计划

> 版本: 1.0  
> 日期: 2026-07-14  
> 状态: 生效

---

## 一、版本号策略 (SemVer)

ConnectAlso 遵循[语义化版本 2.0.0](https://semver.org/lang/zh-CN/)。

```
MAJOR.MINOR.PATCH   例如: 1.2.3
│     │     │
│     │     └─ PATCH: 向后兼容的 bug 修复
│     └─────── MINOR: 向后兼容的新功能
└───────────── MAJOR: 不兼容的 API 变更
```

### 预发布版本

```
1.0.0-alpha.1    Alpha 测试版
1.0.0-beta.1     Beta 测试版
1.0.0-rc.1       Release Candidate
```

## 二、API 稳定性保证

### 控制服务 REST API

| API 版本 | 兼容性承诺 | 说明 |
|----------|:--------:|------|
| `/api/v1/*` | ✅ 稳定 | 当前版本，1.x 全周期维持 |
| 新增端点 | ✅ | MINOR 版本可添加新端点 |
| 已有端点 | ✅ | 不删除、不修改语义 |
| 响应字段 | ✅ | 可添加新字段，不删除已有字段 |
| 请求字段 | ⚠️ | 可添加 optional 字段，不删除 required 字段 |
| `/api/v2/*` | 🔮 未来 | MAJOR 版本可能引入 |

### 二进制 CLI 接口

| 组件 | 稳定性 |
|------|:------:|
| `connectalso status` | ✅ 稳定 |
| `connectalso start` | ✅ 稳定 |
| `connectalso diag` | ✅ 稳定 |
| `connectalso admin *` | ✅ 稳定 |
| `connectalso-daemon --help` 参数 | ⚠️ 可添加，不可删除 |

| 文件 | 兼容性 |
|------|:------:|

## 三、网络协议兼容性

### 协议版本控制

所有网络协议帧包含 `PROTO_VERSION` 字段：

```rust
const PROTO_VERSION: u8 = 0x01;
```

| 变更类型 | 处理方式 |
|----------|----------|
| 同一 PROTO_VERSION | 完全兼容 |
| 版本不匹配 | 拒绝连接 + 日志警告 |
| 新增协议特性 | 增加 MINOR 协议版本号 |

### 数据面加密

```
加密套件锁定: X25519 + ChaCha20-Poly1305
```

密码原语一旦选定，同一 MAJOR 版本内**不更改**。
如需升级（如切换到 Noise 协议），将在 MAJOR 版本中引入并通过
`PROTO_VERSION` 协商。

### 中继协议

```
帧格式: [ver:1][type:1][sender:16][target:16][len:2][payload:N]
```

- 帧头结构不改变
- 新增消息类型为 MINOR 变更
- 移除消息类型为 MAJOR 变更

## 四、支持生命周期

### 版本支持矩阵

```
版本    发布日期      安全更新截止    说明
────    ────────      ────────────    ────
1.0.x   2026-07       2027-07        首个稳定版，12 个月支持
1.1.x   2026-10       2027-10        MINOR 功能更新
2.0.0   2027-07       TBD            下一个 MAJOR（如需要）
```

### 支持策略

| 支持类型 | 时间 | 内容 |
|----------|------|------|
| **完全支持** | 发布后 12 个月 | Bug 修复、安全补丁、功能 backport |
| **安全支持** | 额外 6 个月 | 仅安全补丁，无新功能 |
| **生命周期结束** | 18 个月后 | 不再更新，建议升级 |

### LTS 版本

每 12 个月指定一个 LTS（长期支持）版本：

```
1.0.0  → LTS (18 个月完全支持 + 6 个月安全支持)
1.6.0  → LTS (下一个)
```

## 五、弃用策略

### 弃用流程

```
1. MINOR 版本中标记为 deprecated（日志警告）
2. 文档标注替代方案
3. 至少经过 1 个 MINOR 版本后，在 MAJOR 中移除
```

### 示例

```
v1.1.0: 某参数标记 #[deprecated]，日志输出警告
v1.2.0: 警告升级为 WARN 级别
v2.0.0: 移除该参数
```

### 当前无弃用项

ConnectAlso 1.0.0 无弃用功能。所有 API 和配置项均为全新。

## 六、发布流程

### 发布节奏

| 类型 | 节奏 | 示例 |
|------|------|------|
| PATCH | 按需（安全/严重 bug） | 1.0.0 → 1.0.1 |
| MINOR | 约每 3 个月 | 1.0.0 → 1.1.0 |
| MAJOR | 按需（通常 12-18 个月） | 1.x → 2.0.0 |

### 发布检查清单

- [ ] 所有测试通过 (`cargo test --workspace`)
- [ ] Clippy 无警告 (`cargo clippy --workspace -- -D warnings`)
- [ ] 格式化检查 (`cargo fmt --all -- --check`)
- [ ] 模糊测试通过 (fuzz workflow)
- [ ] 许可证审计 (`cargo deny check licenses`)
- [ ] 安全审计 (`cargo audit`)
- [ ] 更新 CHANGELOG.md
- [ ] 标记 git tag (`v1.0.0`)
- [ ] 构建安装包 (Win/Mac/Linux)
- [ ] GitHub Release 发布
- [ ] Docker 镜像推送
- [ ] 文档更新

### 发布产物

| 平台 | 产物 |
|------|------|
| Linux x86_64 | `.deb`, `.rpm`, `.tar.gz` |
| macOS Universal | `.pkg` |
| Windows x64 | `.msi`, `.zip` |
| Docker | `ghcr.io/lightalso/connectalso:1.0.0` |

## 七、长期维护承诺

### 安全漏洞响应

| 严重性 | 响应时间 | 修复时间 |
|--------|:--------:|:--------:|
| **严重** (RCE/私钥泄露) | 24h | 72h |
| **高** (身份绕过/数据泄露) | 48h | 7d |
| **中** (DoS/信息泄露) | 7d | 30d |
| **低** (理论攻击面) | 30d | 下一 MINOR |

### 依赖更新策略

| 依赖类型 | 更新频率 | 策略 |
|----------|----------|------|
| 密码库 | 立即 | 自动 minor，review major |
| 网络库 | 每月 | CI 自动测试 |
| 工具库 | 按需 | 不阻塞开发 |
| 平台绑定 | 谨慎 | 跨平台测试后更新 |

### 社区承诺

- **Issue 响应**: 3 个工作日内首次回复
- **PR Review**: 1 周内开始 review
- **文档**: 每次 release 同步更新
- **向后兼容**: 最高优先级原则

### 路线图展望

```
2026 Q3  — v1.0.0 (首个稳定版)
2026 Q4  — v1.1.0 (移动端 Beta)
2027 Q1  — v1.2.0 (Noise 协议握手)
2027 Q2  — v1.3.0 (子网路由 / 出口节点)
2027 Q3  — v2.0.0 (如需要不兼容变更)
```
