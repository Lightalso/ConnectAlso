# ConnectAlso 隐私策略

> 最后更新: 2026-07-14

---

## 概述

ConnectAlso 是一个**可自托管**的虚拟局域网工具。当您自行部署服务端时，所有数据完全由您控制。

本文档说明 ConnectAlso 收集、处理和存储哪些数据，以及这些数据的使用方式。

---

## 一、数据分类

### 服务端存储的数据

| 数据类型 | 示例 | 存储位置 | 用途 |
|----------|------|----------|------|
| 设备公钥 | X25519 32 字节 | 数据库 `devices` 表 | 对等身份识别 |
| 设备主机名 | `"alice-laptop"` | 数据库 | 管理标识 |
| 虚拟 IPv4 | `100.64.0.1` | 数据库 | 网络路由 |
| 设备上线时间 | Unix 时间戳 | 数据库 | 在线状态 |
| NAT 候选地址 | `1.2.3.4:5678` | 数据库 `candidates` 表 | P2P 连接辅助 |
| ACL 规则 | 管理员配置 | 数据库 `acl_rules` 表 | 访问控制 |

### 服务端**不**存储的数据

| 数据类型 | 说明 |
|----------|------|
| 设备私钥 | 仅在客户端本地生成和保存 |
| 会话密钥 | 内存中，会话结束即销毁 |
| 业务流量内容 | 中继仅转发密文 |
| 访问令牌/密码 | 不在服务端持久化 |
| DNS 查询日志 | 不记录 |
| 浏览历史 | 不收集 |

### 客户端存储的数据

| 数据类型 | 存储位置 | 保护 |
|----------|----------|------|
| 设备私钥 | 内存（M4 后: Keychain/Keystore） | 进程隔离 |
| 会话密钥 | 内存（`zeroize` 计划 M4） | 会话结束清除 |
| 配置文件 | `~/.config/connectalso/config.json` | 文件权限 |

### 客户端**不**收集的数据

- 不收集遥测/使用统计
- 不包含任何分析 SDK
- 不发送崩溃报告（除非用户手动提交）
- 不进行任何形式的用户追踪

## 二、数据传输

### 控制面 (客户端 ↔ 控制服务)

| 数据 | 加密 | 频率 |
|------|:----:|------|
| 设备注册 | HTTP（建议 TLS） | 首次 + 重启 |
| 心跳 | HTTP | 30s |
| 对等列表 | HTTP | 30s |
| 候选地址 | HTTP | 按需 |
| ACL 规则 | HTTP | 按需 |

> M4 计划添加 TLS 支持，届时所有控制面通信将加密。

### 数据面 (客户端 ↔ 客户端)

| 路径 | 加密 | 说明 |
|------|:----:|------|
| P2P 直连 | ✅ ChaCha20-Poly1305 | 端到端加密 |
| 中继转发 | ✅ ChaCha20-Poly1305 | 中继仅见密文 |

### 中继服务器不参与解密

中继节点仅读取 RelayFrame 协议头（发送/接收 Peer ID），payload 为不透明密文。
中继不具备解密业务流量的能力。

## 三、数据保留与删除

### 自动清理

| 数据类型 | 保留期限 | 清理机制 |
|----------|----------|----------|
| 设备心跳 | 可配置（默认 300s） | `purge_stale` 后台任务 |
| 候选地址 | 可配置 | `purge_stale_candidates` |
| 日志文件 | 每日轮转 | `tracing-appender` |

### 手动删除

```bash
# 注销设备
curl -X DELETE http://localhost:3000/api/v1/register/<device_id>

# 撤销设备（保留记录，释放 IP）
curl -X PUT http://localhost:3000/api/v1/register/<device_id>/revoke

# 删除 ACL 规则
curl -X DELETE http://localhost:3000/api/v1/acl/<rule_id>
```

## 五、自托管说明

ConnectAlso 设计为**完全可自托管**：

- 所有服务端组件可自行部署
- 不依赖任何外部服务或云 API
- 不向第三方发送任何数据
- 设备注册、对等发现、数据转发全部在自建服务端完成

唯一的"外部"通信是 STUN 查询（如果您使用公共 STUN 服务器），
这仅向 STUN 服务器发送 Binding Request 以获取 NAT 映射地址。
